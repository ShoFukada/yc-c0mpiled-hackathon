# プロダクト要件: AI鑑定アシスタント

## コンセプト

スニーカーなどのブランド品の真贋鑑定をAIがアシストするWebアプリ。

鑑定士の専門知識をAIで民主化し、「どこを見るべきか」をガイドする。

## 背景・課題

- フェイク品は巧妙に作られているが、タグ・縫製・生地などに微妙な違いがある
- セカストなどはブランド品買取マニュアルに基づいて店員が鑑定するが、トレーニングコストが高い
- StockX（米国）は専門の鑑定士が全品を検品しているが、人的リソースに依存し非効率
- 鑑定の「見るべきポイント」をAIが提示することで、鑑定プロセスを効率化できる

## ターゲット

- スニーカーの真贋を確認したい個人（メルカリ購入前の確認等）
- セカストの買取担当者
- 鑑定初心者〜中級者

## MVP要件

### 対象

- スニーカー（横からの写真1枚）

### 画面遷移

```
[1. アップロード画面] → [2. ローディング画面] → [3. 鑑定ガイド画面]
```

#### 1. アップロード画面

- スニーカーの写真をアップロード（またはカメラ撮影）
- 横からの写真1枚を受け付ける

#### 2. ローディング画面

- 「鑑定のためのガイドを作成しています...」的な表示
- バックエンドでAI解析中

#### 3. 鑑定ガイド画面（メイン）

**画像エリア（左側/上部）:**
- アップロードされたスニーカー画像を表示
- 見るべきポイントが番号付きでオーバーレイ表示（ピン/マーカー）
- 各ポイントをタップすると、その部分がズームされて詳細が表示される

**ポイント一覧（右側/下部）:**
- 番号付きのテキストリスト（画像上のマーカーと対応）
- 例:
  1. 靴紐 — 編み目の均一性を確認。フェイクは不均一になりやすい
  2. Swooshロゴ — ステッチの始点と終点の位置を確認
  3. ソール接着部 — はみ出しや気泡がないか
  4. タグ — フォント、文字間隔、印刷品質を確認

**インタラクション:**
- ポイントをタップ → 画像のその部分がズーム＋詳細テキスト表示
- ユーザーが実物を見ながら各観点を確認していくフロー

### バックエンド

**入力:**
- スニーカー画像（横からの写真1枚）
- 鑑定のための注目ポイント群（mdテキスト）
  - 鑑定士がメンテナンスしやすいようにMarkdown形式で管理

**処理:**
- AI（Gemini Vision等）が画像を解析
- 注目ポイントmdを解析し、各ポイントを構造化データに整形
- 各ポイントに対応する画像上の座標をマッピング

**出力:**
```json
{
  "points": [
    {
      "id": 1,
      "label": "Swooshロゴ",
      "description": "ステッチの始点と終点の位置を確認。本物は...",
      "bbox": { "x1": 0.35, "y1": 0.45, "x2": 0.55, "y2": 0.65 }
    }
  ]
}
```

## 技術スタック

- Frontend: Vite + React + TanStack Router/Query + Tailwind CSS
- Backend: FastAPI + Python
- AI: Gemini API（Vision）
- データ: 鑑定ポイントをMarkdownファイルで管理

## スコープ外（MVP後）

- 複数アングルの写真対応（「次はここを撮ってください」フロー）
- スニーカー以外のブランド品対応
- 鑑定結果の保存・履歴
- ユーザー認証

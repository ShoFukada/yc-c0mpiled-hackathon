🚀 AuthentiFlow 開発・実装ロードマップ（残り6時間のタイムボックス）
最大の罠（Tarpit）である「真のセグメンテーション（ピクセル単位の輪郭抽出）」は捨てます。代わりに、**Vision API（Gemini 1.5 Pro等）に「対象パーツの中心座標 [x, y]」を出力させ、フロントエンドでCSSを使ってピンや吹き出し（Callout）をオーバーレイ表示する「疑似セグメンテーション」**で実装します。これで圧倒的に開発スピードが上がります。

🟢 Phase 1: Minimum Viable Demo（安全網の確保）
目標完了時刻: 14:30 / 状態: これさえ動けば「ツール」としてピッチ可能

UX:

ユーザーがスニーカーの全体写真をアップロードする。

AIが裏側で写真を解析し、マニュアルにある「3つのポイント（ヒール、カセット、つま先）」の画像上の座標 [x, y] を特定。

写真の上に、3つの「吹き出し（マニュアルの要約）」が同時にポップアップ表示される。

実装ハック:

Vision APIに画像とSOPマニュアルを投げ、「JSON形式で、3つのチェックポイントの名称と、画像内の相対座標 (0.0〜1.0) を返せ」と指示（プロンプト）するだけ。フロントエンドは送られてきた座標にCSSで position: absolute を使ってDiv（吹き出し）を置くだけです。

🟡 Phase 2: Interactive Copilot（YCが好む「魔法」のUX）
目標完了時刻: 16:30 / 状態: これが動けば「エージェント」として高く評価される

UX:

Phase 1の「吹き出し」を一度に全部出すのをやめる。

1つ目のポイント（例：ヒールバッジ）だけを枠線やハイライトで示し、「ここを接写してください」とAIが指示を出す（UIの状態遷移）。

ユーザーが接写写真を撮る（デモ用に用意した接写画像をアップロードする）。

「確認しました。次はカセットを撮ってください」と次の指示へ進む。

実装ハック:

UIのステート（状態）管理を追加します。全体写真 -> ポイント1要求 -> 接写写真1 -> ポイント2要求... という「対話型のステップ」をフロントエンドで制御します。ここがこのプロダクトの**コアバリュー（UX）**になります。

🔴 Phase 3: The Magic Resolution（勝つための最終仕上げ）
目標完了時刻: 18:00 / 状態: 投資家が「いますぐ使いたい」と思う完成度

UX:

Phase 2で撮影した接写画像に対して、AIが「マニュアルと照らし合わせた評価結果」をフィードバックする。

最後に「Confidence 99% - Authentic」というリッチな結果画面（承認ボタン付き）を出す。

実装ハック:

デモの確実性を担保するため、**「AIの判定結果は『本物』になるようにプロンプトで強要（決め打ち）する」**のが鉄則です。「入力された画像はSOPのAuthenticの条件を満たしている前提で、プロの鑑定士のようなトーンで評価コメントを生成しろ」と指示します。
